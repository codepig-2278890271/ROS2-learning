# 自定义接口

## 为什么要自己定义接口

前面你一直在用 ROS2 自带的接口：

```python
from example_interfaces.msg import String        # 话题
from example_interfaces.srv import AddTwoInts     # 服务
from example_interfaces.action import Fibonacci   # 动作
```

但实际项目里数据不是「字符串」和「整数」能打发的：

```
要发「机器人状态」 → 需要 RobotStatus.msg（电量、位置、速度、温度）
要查询「最近充电桩」→ 需要 FindCharger.srv（请求当前位置 → 返回充电桩列表）
要让机器人导航     → 需要 NavigateTo.action（目标位姿 → 进度 → 到达确认）
```

**自定义接口就是你自己设计的通信协议。**

---

## 三大接口文件

| 文件 | 用在 | 类比 |
|------|------|------|
| `.msg` | 话题通信 | 定义一个结构体 |
| `.srv` | 服务通信 | 定义请求 + 响应 |
| `.action` | 动作通信 | 定义目标 + 结果 + 反馈 |

### 语法速览

```
# ━━━ .msg（消息）━━━
类型 字段名 注释

# ━━━ .srv（服务）━━━
请求字段
---
响应字段

# ━━━ .action（动作）━━━
目标字段
---
结果字段
---
反馈字段
```

---

## 关键决策：接口放哪

ROS2 的接口（`.msg` / `.srv` / `.action`）需要**代码生成**——从定义文件生成 Python 类。这个生成过程用的是 CMake 工具链，所以**纯 Python 包不能包含接口定义**。

你有两种方案：

| 方案 | 做法 | 适用场景 |
|------|------|---------|
| A. 独立接口包（CMake） | 新建一个 CMake 包，只放接口定义 | **推荐**，接口和实现分离 |
| B. 混合包（CMake + Python） | 一个包既是 CMake 又装 Python | 小项目，接口和代码放一起 |

> 官方最佳实践是方案 A——接口包独立，其他 Python 包依赖它。这样接口是纯 C 编译产物，任何语言都能用。

本笔记用方案 A，建一个 `learning_interface` 包专门放接口。

## 第一步：创建接口包

```bash
cd ~/ROS2-Learning/ros2_ws/src

# 注意：用 ament_cmake，不是 ament_python
ros2 pkg create learning_interface \
  --build-type ament_cmake \
  --dependencies rosidl_default_generators \
  --license "TODO"
```

建好后的目录结构：

```
learning_interface/
├── msg/            ← 新建，放 .msg 文件
├── srv/            ← 新建，放 .srv 文件
├── action/         ← 新建，放 .action 文件
├── CMakeLists.txt
└── package.xml
```

创建子目录：

```bash
mkdir -p learning_interface/msg learning_interface/srv learning_interface/action
```

## 第二步：写接口定义

### 2.1 自定义消息：RobotStatus.msg

`learning_interface/msg/RobotStatus.msg`：

```
# 机器人状态消息
string robot_name          # 机器人名字
float32 battery_level      # 电量百分比 0.0~1.0
float32 temperature        # 温度（℃）
float32 position_x         # 当前位置 X
float32 position_y         # 当前位置 Y
float32 velocity           # 当前速度 m/s
bool is_charging           # 是否在充电中
```

### 2.2 自定义服务：FindCharger.srv

`learning_interface/srv/FindCharger.srv`：

```
# 请求：当前位置
float32 current_x
float32 current_y
---
# 响应：最近的充电桩
float32 charger_x
float32 charger_y
float32 distance
bool success
string message
```

### 2.3 自定义动作：NavigateTo.action

`learning_interface/action/NavigateTo.action`：

```
# 目标：目标位姿
float32 target_x
float32 target_y
float32 target_yaw
---
# 结果：是否到达
bool reached
string message
---
# 反馈：导航中的进度
float32 current_x
float32 current_y
float32 distance_remaining     # 还剩多少米
int32 percent_complete         # 完成百分比
```

### 字段类型一览

| 类型 | 说明 |
|------|------|
| `bool` | 布尔 |
| `int8` `int16` `int32` `int64` | 有符号整数 |
| `uint8` `uint16` `uint32` `uint64` | 无符号整数 |
| `float32` `float64` | 浮点数 |
| `string` | 字符串 |
| `type[]` | 数组，如 `int32[]` `string[]` |
| `type[N]` | 定长数组，如 `float32[3]` |
| `Header header` | ROS 消息头（时间戳 + 坐标系） |

> 默认值：数字 = 0，字符串 = ""，bool = false，数组 = 空。

## 第三步：配置 CMakeLists.txt

`learning_interface/CMakeLists.txt`：

```cmake
cmake_minimum_required(VERSION 3.8)
project(learning_interface)

# ━━━ ① 引入代码生成器 ━━━
find_package(rosidl_default_generators REQUIRED)

# ━━━ ② 列出所有接口文件 ━━━
set(msg_files
  "msg/RobotStatus.msg"
)

set(srv_files
  "srv/FindCharger.srv"
)

set(action_files
  "action/NavigateTo.action"
)

# ━━━ ③ 生成代码 ━━━
rosidl_generate_interfaces(${PROJECT_NAME}
  ${msg_files}
  ${srv_files}
  ${action_files}
  DEPENDENCIES
)

# ━━━ ④ 安装 ━━━
ament_package()
```

每条新增的接口都要在这里登记，少写一个就不会生成。

## 配置 package.xml

`learning_interface/package.xml`：

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd"
  schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>learning_interface</name>
  <version>0.0.0</version>
  <description>学习用的自定义接口包</description>
  <maintainer email="your@email.com">zhuzeyu</maintainer>
  <license>TODO</license>

  <!-- 构建时依赖代码生成器 -->
  <buildtool_depend>ament_cmake</buildtool_depend>
  <buildtool_depend>rosidl_default_generators</buildtool_depend>

  <!-- 运行时依赖：接口里用到的其他接口包 -->
  <!-- 如果没用到其他包的接口，这行可以删掉 -->

  <!-- 让其他包能发现这些接口 -->
  <member_of_group>rosidl_interface_packages</member_of_group>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

> 关键：`<member_of_group>rosidl_interface_packages</member_of_group>` —— 告诉 ROS2「这是个接口包」，工具链才能正确索引。

## 第五步：在 Python 代码里用自定义接口

### 发布自定义消息

```python
import rclpy
from rclpy.node import Node
from learning_interface.msg import RobotStatus   # ← 自己的消息！


class StatusPublisher(Node):

    def __init__(self):
        super().__init__('status_publisher')
        self._publisher = self.create_publisher(
            RobotStatus, 'robot_status', 10       # ← 用自己的消息类型
        )
        self._timer = self.create_timer(1.0, self.publish_status)
        self._counter = 0

    def publish_status(self):
        msg = RobotStatus()
        msg.robot_name = 'turtlebot_01'
        msg.battery_level = 0.85
        msg.temperature = 42.3
        msg.position_x = float(self._counter)
        msg.position_y = 0.0
        msg.velocity = 0.5
        msg.is_charging = False

        self._publisher.publish(msg)
        self.get_logger().info(f'📡 已发布: {msg.robot_name} @ ({msg.position_x}, {msg.position_y})')
        self._counter += 1
```

### 调用自定义服务

```python
from learning_interface.srv import FindCharger


class ChargerClient(Node):

    def __init__(self):
        super().__init__('charger_client')
        self._client = self.create_client(FindCharger, 'find_charger')

    def find_nearest_charger(self, x, y):
        req = FindCharger.Request()
        req.current_x = x
        req.current_y = y
        return self._client.call_async(req)
```

### 使用自定义动作

```python
from learning_interface.action import NavigateTo


# 和用 Fibonacci 一模一样，只是换成了自己的 NavigateTo
class NavClient(Node):

    def __init__(self):
        super().__init__('nav_client')
        self._client = ActionClient(self, NavigateTo, 'navigate_to')

    def send_goal(self, x, y, yaw):
        goal = NavigateTo.Goal()
        goal.target_x = x
        goal.target_y = y
        goal.target_yaw = yaw
        # ... 后续和 Fibonacci 动作客户端一样
```

### 让 Python 包依赖你的接口包

你的 `learning_topic` 或新包要加依赖关系：

**package.xml** 加：

```xml
<depend>learning_interface</depend>
```

**setup.py** 不需要改，依赖信息在 package.xml 里声明就行。

## 接口命名惯例

| 类别 | 命名风格 | 示例 |
|------|---------|------|
| `.msg` | 大驼峰 | `RobotStatus`, `LaserScan`, `PointCloud` |

## 常见类型速查

### 基本类型

```
bool     int32    float32    string    uint8
```

### 数组

```
int32[] numbers         → [1, 2, 3]
string[] names          → ["a", "b"]
float32[3] position     → [0.0, 0.0, 0.0]  定长3个
```

### Header（强烈推荐在消息里加）

```
std_msgs/Header header  # 自动获得时间戳和坐标系
int32 data
```

Header 包含：
- `header.stamp` — 时间戳（消息什么时候产生的）
- `header.frame_id` — 坐标帧（数据在哪个坐标系下）

**几乎所有传感器消息都带 Header**——下一章 TF2 就会用到。

### 嵌套自定义类型

```msg
# MoveCommand.msg
string robot_name
float32 speed
float32 duration
```

```msg
# MoveSequence.msg
MoveCommand[] commands   # ← 用你刚才定义的 MoveCommand！
```

## 总结

```
自定义接口 = 自己设计的通信协议

三步走：
  ① 创建 CMake 接口包（ros2 pkg create --build-type ament_cmake）
  ② 写 .msg / .srv / .action 文件
  ③ CMakeLists.txt 注册 → colcon build → source → 直接用

Python 包里用：
  from learning_interface.msg import RobotStatus    ← 像用标准库一样
  from learning_interface.srv import FindCharger
  from learning_interface.action import NavigateTo

关键坑：
  - 接口包必须是 CMake，不能是纯 Python
  - package.xml 要加 <member_of_group>rosidl_interface_packages</member_of_group>
  - CMakeLists.txt 里少登记一个文件就少生成一个接口
  - 新增 .action 文件后记得加到 CMakeLists.txt 和 package.xml
```

| | 话题 | 服务 | 动作 |
|------|------|------|------|
| 定义文件 | `.msg` | `.srv` | `.action` |
| 结构 | 字段列表 | 请求 `---` 响应 | 目标 `---` 结果 `---` 反馈 |
| Python 导入 | `from xxx.msg import Yyy` | `from xxx.srv import Yyy` | `from xxx.action import Yyy` |
| CLI 查看 | `ros2 interface show` | 同上 | 同上 |

自定义接口让你的代码从「凑合能用 example_interfaces」升级到「正儿八经的机器人通信协议」。
