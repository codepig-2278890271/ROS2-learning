# 总结与常用命令速查

## 知识全景图

学完了前九章，你已经掌握了 ROS2 核心基础设施。用一张图串起来：

```
                         ┌──────────────┐
                         │   Launch 文件  │  ← 一键启动整个系统
                         │  (ros2 launch) │
                         └──────┬───────┘
                                │ 启动并配置
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
   ┌────▼─────┐          ┌──────▼──────┐         ┌──────▼──────┐
   │   参数    │          │    节点      │         │  自定义接口  │
   │Parameter │──────────▶   Node      │◀────────│ .msg .srv   │
   │ 运行时配置 │  配置节点  │  计算单元    │ 通信协议 │  .action    │
   └──────────┘          └──────┬──────┘         └─────────────┘
                                │
             ┌──────────────────┼──────────────────┐
             │                  │                  │
        ┌────▼────┐      ┌─────▼─────┐      ┌─────▼─────┐
        │ 话题通信 │      │  服务通信   │      │  动作通信   │
        │  Topic  │      │  Service   │      │  Action    │
        │ 单向广播 │      │  一问一答   │      │ 长时间任务  │
        └─────────┘      └───────────┘      └───────────┘
```

**一句话总结**：用自定义接口（09）定义通信协议，节点（03）通过话题/服务/动作（04-06）互相通信，参数（07）让节点可配置，Launch 文件（08）一键启动整个系统。

## 三大通信机制决策表

这是你最常问自己的问题：**「这个场景用什么？」**

```
                需要回复吗？
               /          \
             否            是
             │             │
         持续/高频？    耗时久 / 需要反馈 / 可中途取消？
         /      \      /          \
       是        否   否           是
       │         │    │            │
     Topic     少见 Service      Action

具体判断：
  Topic    → 传感器数据、状态广播、持续数据流（激光雷达、图像、里程计）
  Service  → 查询、配置指令、瞬时操作（获取地图、设置模式、拍一张照）
  Action   → 导航、抓取、长时间计算、任何需要反馈+可取消的任务
```

## 日常开发 CLI 速查

### 工作空间

```bash
# 创建并编译
mkdir -p ~/ros2_ws/src && cd ~/ros2_ws
colcon build                    # 全编
colcon build --packages-select 包名  # 只编一个包
colcon build --symlink-install  # 符号链接安装（Python 改代码不用重编！）

# 环境
source install/setup.bash        # 每个新终端都要 source
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc  # 自动 source
```

### 包管理

```bash
ros2 pkg list                                # 列出所有已安装的包
ros2 pkg list | grep learning                # 只看自己的包
ros2 pkg create 包名 --build-type ament_python  # 创建 Python 包
ros2 pkg create 包名 --build-type ament_cmake --dependencies rosidl_default_generators  # 创建接口包
ros2 pkg executables 包名                     # 看一个包有哪些可执行文件
```

### 节点

```bash
ros2 node list                     # 当前有哪些节点在跑
ros2 node info /节点名              # 节点的订阅/发布/服务/动作完整信息
ros2 run 包名 可执行文件名           # 启动一个节点
```

### 话题

```bash
ros2 topic list                    # 列出所有话题
ros2 topic info /话题名             # 话题的类型、发布者、订阅者
ros2 topic echo /话题名             # 实时打印消息内容（调试利器）
ros2 topic hz /话题名               # 看发布频率
ros2 topic pub /话题名 类型 "{...}"  # 手动发一条消息
```

### 服务

```bash
ros2 service list                  # 列出所有服务
ros2 service type /服务名           # 查看服务类型
ros2 service call /服务名 类型 "{...}"  # 手动调用
ros2 service find 类型              # 找提供该类型服务的服务端
```

### 动作

```bash
ros2 action list                   # 列出所有动作
ros2 action info /动作名            # 查看动作信息
ros2 action send_goal /动作名 类型 "{...}"  # 发目标（自动打印反馈+结果）
```

### 参数

```bash
ros2 param list                    # 列出节点参数
ros2 param get /节点名 参数名        # 查看参数值
ros2 param set /节点名 参数名 值     # 运行时修改参数
ros2 param dump /节点名             # 导出参数（可用于写 YAML 文件）
ros2 param load /节点名 文件.yaml    # 从文件加载参数
```

### Launch

```bash
ros2 launch 包名 launch文件名                    # 启动 launch 文件
ros2 launch 包名 launch文件名 参数:=值            # 带参数启动
ros2 launch 包名 launch文件名 --show-args        # 看有哪些可配参数
ros2 launch --debug 包名 launch文件名             # 调试模式，打印更详细日志
```

### 接口

```bash
ros2 interface list                # 列出所有接口
ros2 interface show 完整名称        # 查看接口结构
ros2 interface list | grep 你的包名  # 验证自定义接口是否生成
ros2 interface proto 完整名称       # 看原始定义文件内容
```

## 调试流程速查

```
问题：节点间不通信？

  ros2 node list                         ① 节点都在跑吗？
  ros2 node info /节点名                  ② 节点的 发布/订阅/服务 对吗？
  ros2 topic list                        ③ 话题名拼写一致吗？
  ros2 topic info /话题名                 ④ 发布者和订阅者类型匹配吗？
  ros2 topic echo /话题名                 ⑤ 有数据在发吗？内容对吗？
  ros2 topic hz /话题名                   ⑥ 频率对吗？

问题：服务调用失败？

  ros2 service list                      ① 服务存在吗？
  ros2 service type /服务名               ② 类型对吗？
  ros2 service call /服务名 类型 "{...}"   ③ 手动能调通吗？（先排除客户端代码问题）

问题：动作跑不起来？

  ros2 action list                       ① 动作存在吗？
  ros2 action info /动作名                ② 服务端连着吗？
  ros2 action send_goal ...              ③ 命令行能发目标吗？（先排除客户端代码问题）

问题：节点启动就崩？

  ros2 run 包名 可执行文件                ① 直接跑看报什么错
  colcon build --symlink-install         ② 装了 --symlink-install 吗？改了代码要重编
  source install/setup.bash              ③ source 过了吗？
```

## setup.py 的标准写法

```python
import os
from glob import glob
from setuptools import find_packages, setup

package_name = '你的包名'

setup(
    name=package_name,
    version='0.0.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        ('share/ament_index/resource_index/packages', ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
        # 有 launch 文件就加这段：
        (os.path.join('share', package_name, 'launch'), glob('launch/*_launch.py')),
        # 有 YAML 配置文件就加这段：
        (os.path.join('share', package_name, 'config'), glob('config/*.yaml')),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='zhuzeyu',
    maintainer_email='your@email.com',
    description='包的描述',
    license='TODO',
    entry_points={
        'console_scripts': [
            '可执行文件名 = 包名.模块名:main函数名',
        ],
    },
)
```

## 包的依赖声明

在 `package.xml` 里加依赖：

```xml
<depend>rclpy</depend>               <!-- Python 客户端库，必加 -->
<depend>std_msgs</depend>            <!-- 用了 String 等标准消息 -->
<depend>example_interfaces</depend>  <!-- 用了 AddTwoInts, Fibonacci -->
<depend>learning_interface</depend>  <!-- 用了你自己写的接口 -->
```

> 加了 `<depend>` 才能 `find_package()` 并 `from xxx import yyy`。忘了加 → 编译报 `No module named xxx`。

## 最易踩的六个坑

### 1. 忘了 source

```bash
# 每个新终端都要 source！常见症状：
# - ros2: command not found
# - import rclpy 报错
# - ros2 run 找不到包

source /opt/ros/humble/setup.bash      # ROS2 本体
source ~/ros2_ws/install/setup.bash    # 你自己的工作空间
```

### 2. 话题名 / 服务名不匹配

```python
# 发布者
create_publisher(String, 'number_topic', 10)

# 订阅者 → 名字不一样，收不到！
create_subscription(String, 'number_tpoic', ...)  # ❌ 拼错了
```

排查：`ros2 topic list` 看到只有一个有话题，另一个没有 → 名字拼错了。

### 3. 消息字段名写错

```python
msg = String()
msg.data = 'hello'       # ✅
msg.date = 'hello'       # ❌ 字段名不存在
msg.data = 123           # ❌ 类型不对，String.data 要字符串
```

排查：`ros2 interface show std_msgs/msg/String` 看字段名叫什么。

### 4. 接口包没用 CMake

```bash
# ❌ 错误：用 ament_python 建接口包
ros2 pkg create my_interface --build-type ament_python

# ✅ 正确：用 ament_cmake
ros2 pkg create my_interface --build-type ament_cmake --dependencies rosidl_default_generators
```

只有 CMake 包才能编译 `.msg` `.srv` `.action`。

### 5. Python 代码改了不生效

```bash
# 没加 symlink-install？每次改代码都要重编
colcon build --packages-select 你的包名

# 加了 --symlink-install？改 .py 文件直接生效，但以下情况要重编：
# - 改了 setup.py（新增 entry_points）
# - 改了 package.xml（新增依赖）
# - 改了 launch / config 文件位置

# 强烈建议编译时永远加 --symlink-install
colcon build --symlink-install
```

### 6. spin 的线程问题

```python
# ❌ 在回调里写死循环或 time.sleep()
def callback(self, msg):
    time.sleep(5)  # 卡死整个 spin！所有回调都停摆

# ✅ 耗时操作放独立线程或 Timer
def callback(self, msg):
    thread = threading.Thread(target=self.heavy_work, args=(msg,))
    thread.start()
```

唯一例外：Action 的 `execute_callback` 自带独立线程，可以 `time.sleep()`。

## 项目实战：从零搭建一个 ROS2 系统的标准流程

给你一个可复用的开发顺序：

```
① 设计接口（先想清楚节点间传什么、怎么传）
   → 创建 learning_interface（CMake 包）
   → 写 .msg / .srv / .action
   → 编译验证：ros2 interface show xxx

② 逐个实现节点
   → 创建 Python 包
   → 写节点代码（先声明参数，再创建 pub/sub/srv/action）
   → 加 entry_points 到 setup.py
   → 编译验证：ros2 run xxx

③ 给节点加参数
   → declare_parameter + 参数回调校验
   → 用 ros2 param set 手动测 → 确认参数能控制行为

④ 写 YAML 配置文件（可选）
   → config/xxx.yaml
   → 启动时 --ros-args --params-file config/xxx.yaml

⑤ 写 Launch 文件
   → 设定参数、话题重映射
   → 一个 ros2 launch 全部起来
   → 加 DeclareLaunchArgument 让 launch 文件可配置

⑥ 端到端测试
   → ros2 launch xxx
   → ros2 topic echo /xxx 看数据
   → ros2 service call /xxx 测服务
   → ros2 action send_goal /xxx 测动作
```

```
  pub = create_publisher(类型, '话题名', 队列)

  cli = ActionClient(self, 类型, '动作名')

  get_parameter('键').value

  CMake 包 + msg/srv/action/ 目录 + rosidl_generate_interfaces

  source install/setup.bash

─ 排错三板斧 ────────────────────────────────────

  ② 话题名 / 服务名拼对了吗？
  ③ colcon build 过了吗？
```
