# 什么是 ROS2

## 一句话概括

**ROS2**（Robot Operating System 2）是一个为机器人开发提供**通信框架 + 工具集 + 库生态**的开源中间件。它帮你解决“机器人上十几个程序怎么互相通信”这个核心问题。

> ROS2 不是传统意义上的操作系统（比如 Linux、Windows），而是一套**分布式通信中间件**，运行在 Linux/Windows/macOS 之上。

---

## 为什么要用 ROS2？

每种机器人、每个传感器、每个算法都不同。没有 ROS2，你每次都要从头解决：

- 程序之间怎么传数据？（共享内存？Socket？管道？）
- 新加一个程序，怎么让其他程序发现它？
- 程序之间语言不同怎么办？（C++ 和 Python 怎么通信？）
- 怎么记录、回放、可视化数据？

ROS2 提供了一套**标准答案**，让你专注写机器人的业务逻辑，不用重复造轮子。

---

## ROS2 的系统架构

ROS2 采用**分层架构**，自底向上每一层只依赖下一层：

```
┌─────────────────────────────────────────┐
│             用户应用层                    │  ← 你写的机器人程序
├─────────────────────────────────────────┤
│         rcl (ROS 客户端库)               │  ← rclpy(Python) / rclcpp(C++)
├─────────────────────────────────────────┤
│          rmw (ROS 中间件接口)             │  ← 抽象层，解耦底层通信实现
├─────────────────────────────────────────┤
│           DDS (数据分发服务)              │  ← 实际干活：节点发现、消息收发、QoS
├─────────────────────────────────────────┤
│          操作系统 (Linux/Windows/macOS)   │  ← 底层 OS
└─────────────────────────────────────────┘
```

**逐层解释：**

| 层 | 定位 | 你需要关心？ |
|----|------|------------|
| **用户应用层** | 你写的机器人程序（感知、规划、控制等） | ✅ 你的主战场 |
| **rcl** | ROS2 的 API，提供创建节点、发布/订阅消息等编程接口。Python 用 `rclpy`，C++ 用 `rclcpp` | ✅ 写代码天天用 |
| **rmw** | ROS 中间件接口，把「ROS2 概念」（话题、服务）翻译成底层 DDS 操作。让 ROS2 不绑定某个具体 DDS 实现 | ❌ 透明，不感知 |
| **DDS** | Data Distribution Service，工业级分布式通信协议。负责节点自动发现、消息的收发、QoS（服务质量）策略 | ❌ 默认配置够用 |
| **OS** | 底层操作系统 | ❌ |

---

### 为什么 ROS2 选择 DDS？

```
ROS1 的问题：
  - 自己实现的通信协议（TCPROS/UDPROS）
  - 依赖一个中心节点 ROS Master（Master 挂了，整个系统瘫痪）
  - 实时性差
  - 跨平台支持弱

ROS2 的答案：用 DDS
  - DDS 是成熟的工业标准（航空航天、军事、工业控制都在用）
  - 去中心化：没有 Master，节点之间自动发现、直连通信
  - 实时性好：支持 QoS 配置（可靠性、延迟、带宽等可调节）
  - 跨平台：Linux / Windows / macOS 都支持
```

> 简单类比：ROS1 像电话交换机（所有通话经过中心转接），ROS2 像对讲机（点对点直连，自动搜频）。

## 安装 ROS2

> 当前系统：Ubuntu 22.04 → **ROS2 Humble Hawksbill**（LTS，支持到 2027 年 5 月）

### 鱼香 ROS 一键安装

鱼香 ROS（fishros.org.cn）把繁琐的安装步骤整合成了一条命令，对国内用户非常友好：

```bash
source <(wget -qO- http://fishros.com/install)
```

这条命令自动完成五件事：

| 步骤 | 做的事 |
|------|--------|
| ① 换源 | 把 apt 源替换为国内镜像，下载速度飞起 |
| ② 安装 ROS2 | 自动检测 Ubuntu 版本，安装对应的桌面完整版（含 RViz、rqt、turtlesim） |
| ③ 配环境 | 自动把 `source /opt/ros/humble/setup.bash` 写入 `~/.bashrc` |
| ④ 装编译工具 | 安装 `colcon`、`rosdepc` 等构建依赖 |
| ⑤ 配 rosdepc | 配置国内版 `rosdepc`，解决 `rosdep update` 在国内网络下的连接问题 |

> `rosdep` 是 ROS 的依赖管理工具，根据 `package.xml` 自动安装系统依赖。`rosdepc` 是鱼香提供的国内优化版，解决 GitHub 访问不稳定的问题。

### 验证安装

```bash
ros2 run turtlesim turtlesim_node
```

弹出一个蓝色背景的小乌龟窗口 → 安装成功 🎉

---

## 总结

> **ROS2 是一套分布式通信中间件**，分层架构清晰：应用层 → rcl → rmw → DDS → OS。它去中心化、支持多种编程语言、跨平台，让机器人开发者可以专注业务逻辑而不是通信底层。
