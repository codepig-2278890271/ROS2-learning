# 节点（Node）

## 什么是节点

节点是 ROS2 里**最小的运行单元**。一个节点 = 一个独立的 Python 进程，通常只干一件事。

举个例子，一个机器人任务拆成节点：

```
/lidar_driver     → 读激光雷达数据，发布到 /scan 话题
/camera_driver    → 读摄像头数据，发布到 /image 话题
/slam             → 订阅 /scan，做建图，发布 /map
/path_planner     → 订阅 /map，做路径规划，发布 /path
/motor_controller → 订阅 /path，控制电机转动
```

每个节点可以单独启动、单独调试、单独重启。一个节点挂了不影响其他节点。

> **文件 ≠ 节点。** 节点是运行时的 `Node` 对象，不是 `.py` 文件。一个 `.py` 文件里可以写多个节点类，一个 `main()` 里也可以创建多个 `Node` 实例。只不过初学阶段，我们习惯一个 entry_point → 一个 `main()` → 一个节点，这是最常见的模式。

## 一个最小节点

新建 `learning_node/learning_node/my_first_node.py`：

```python
import rclpy
from rclpy.node import Node


def main():
    rclpy.init()                                  # ① 初始化 ROS2 客户端
    node = Node('my_first_node')                  # ② 创建节点，取名
    rclpy.spin(node)                              # ③ 让节点持续运行
    node.destroy_node()                           # ④ 销毁节点
    rclpy.shutdown()                              # ⑤ 关闭 ROS2 客户端


if __name__ == '__main__':
    main()
```

### 逐行解释

```python
rclpy.init()
```

初始化 ROS2 的 Python 客户端。整个进程只需调一次，相当于「开机」。不调这行，后面的 `Node()` 会报错。

```python
node = Node('my_first_node')
```

创建一个节点，名字是 `'my_first_node'`。这个名字会出现在 `ros2 node list` 里。

```python
rclpy.spin(node)
```

让节点进入**死循环**，持续监听话题消息、服务请求等。`spin` 在节点没收到消息时处于休眠状态，不占 CPU。没有这一行，节点创建完就立刻退出了。

```python
node.destroy_node()
rclpy.shutdown()
```

`spin` 结束后（比如 Ctrl+C），手动清理资源。`destroy_node` 注销节点，`shutdown` 关闭 ROS2 客户端。

## 注册可执行入口

写完代码还不行，`ros2 run` 找不到你的节点。需要去 `setup.py` 注册：

```python
entry_points={
    'console_scripts': [
        'my_node = learning_node.my_first_node:main',
    ],
},
```

| 字段 | 含义 |
|------|------|
| `my_node` | 命令名，`ros2 run learning_node my_node` 用的就是这个名 |
| `learning_node` | 包名（内层 Python 包） |
| `my_first_node` | 文件名（`my_first_node.py`），不带 `.py` |
| `main` | `my_first_node.py` 里的入口函数名 |

## 编译并运行

```bash
cd ~/ros2_ws
colcon build --symlink-install
source install/setup.bash

ros2 run learning_node my_node
```

开了之后看不出效果，因为这只是个空节点，什么都没做。另开一个终端试试：

```bash
ros2 node list          # 能看到 /my_first_node
ros2 node info /my_first_node   # 查看节点的详细信息
```

## 常用命令

```bash
ros2 node list                          # 列出所有运行中的节点
ros2 node info /my_first_node           # 查看某个节点的发布/订阅/服务
ros2 run <包名> <可执行命令名>           # 启动某个节点
ros2 run <包名> <命令名> --ros-args \
  -r __node:=新名字                      # 启动节点并改名
```

### 运行时改名

如果你用 `ros2 run` 启动了同一个节点两次，ROS2 会自动给后面那个加后缀 `_1`：

```
/my_first_node
/my_first_node_1
```

也可以显式改名：

```bash
ros2 run learning_node my_node --ros-args -r __node:=my_renamed_node
```

## OOP 写法（推荐）

上面是最小示例，实际开发一般用面向对象写法，把节点逻辑封装成一个类：

```python
import rclpy
from rclpy.node import Node


class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')     # 调用父类 Node 的构造函数，传节点名


def main():
    rclpy.init()
    node = MyNode()                     # 实例化你的节点类
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

继承 `Node` 类，你的自定义逻辑（创建发布者、订阅者、定时器）全部写在 `__init__` 里。这是 ROS2 Python 的**标准写法**，后面的发布者和订阅者都会用这种模式。

## 总结

```
最小节点五部曲：
  rclpy.init()     → 开机
  Node('名字')      → 创建节点
  rclpy.spin(node) → 循环运行
  destroy_node()   → 销毁节点
  rclpy.shutdown() → 关机
```

- 一个 `.py` 文件 = 一个节点
- `setup.py` 的 `entry_points` 是桥梁，让 `ros2 run` 找到你的 `main()` 函数
- 推荐 OOP 写法，继承 `Node`，逻辑写 `__init__` 里
- 空节点虽然不干活，但能验证「创建 → 编译 → 运行 → ros2 node list 看到」这条链路是否通畅
