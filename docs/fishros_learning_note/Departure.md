# ROS2 介绍

## 1. ROS2 的核心思想

ROS2 的核心是“通信 + 组件化”。

- 通过节点（Node）封装功能模块
- 通过话题（Topic）、服务（Service）、动作（Action）等通信机制连接节点
- 采用 DDS 作为底层中间件，实现进程间、跨主机、跨语言的可靠数据传输

因此，ROS2 更像一个分布式通信框架，而不是单纯的机器人库。

## 2. ROS2 的分层架构

```
应用层
ROS2 客户端库层
DDS 抽象层（RMW）
DDS 实现层
操作系统层
```

### 2.1 操作系统层

- 管理进程、线程、网络、内存
- 提供基础运行环境
- ROS2 本身依赖操作系统提供的进程通信、网络接口等能力

### 2.2 DDS 实现层

- 由具体的 DDS 库提供，如 Fast DDS、Cyclone DDS、RTI Connext 等
- 负责实际消息发现、数据收发、QoS 控制、跨进程/跨设备通信
- 这里就是 ROS2 的“中间件实现”层

### 2.3 DDS 抽象层（RMW）

- RMW 全称为 ROS MiddleWare interface
- 定义 ROS2 与底层 DDS 之间的统一接口
- 由 ROS2 的 rcl 层调用，屏蔽不同 DDS 实现的差异
- 让开发者不需要绑定某一种 DDS，实现可插拔替换

> 可以简单理解为：DDS 层是中间件实现，RMW 是 ROS2 与中间件之间的适配层。

### 2.4 ROS2 客户端库层

- 提供节点、话题、服务、动作、参数、计时器等 API
- 统一封装 ROS2 核心功能，供用户代码调用
- 主要组成：
  - `rcl`：ROS Client Library 核心，C 语言实现
  - `rclcpp`：C++ 客户端库
  - `rclpy`：Python 客户端库

### 2.5 应用层

- 用户的机器人业务代码
- 业务逻辑、控制算法、传感器处理、任务规划等
- 基于 ROS2 客户端库进行开发

## 3. ROS2 的通信模型

### 3.1 话题（Topic）

- 发布/订阅机制
- 适合一对多、广播式的数据流
- 常用于传感器数据、状态信息、命令数据等

### 3.2 服务（Service）

- 请求/响应模式
- 适合同步调用、短时交互
- 常用于配置读取、命令执行、状态查询等

### 3.3 动作（Action）

- 适合异步、可预期的长时间任务
- 支持目标发送、反馈和结果
- 常用于导航、运动规划、执行任务等

## 4. QoS 与可靠性

ROS2 继承 DDS 的 QoS 概念，允许用户根据场景选择不同的策略：

- 可靠性（Reliability）
- 持久性（Durability）
- 压力窗口（History）
- 生存时间（Liveliness）
- 目的地顺序（Deadline / Lifespan）

这使得 ROS2 可以更好地适应实时、无线网络、分布式系统等复杂场景。

## 5. 常用工具与调试

### 5.1 命令行工具

- `ros2 topic list`
- `ros2 topic echo`
- `ros2 node list`
- `ros2 service list`
- `ros2 action list`
- `ros2 param set/get`
- `ros2 launch`

### 5.2 可视化与调试工具

- `rviz2`：可视化传感器、坐标系、导航信息等
- `rqt`：图形化插件框架，包含主题监视、参数调试、节点状态
- `ros2 doctor`：检查 ROS2 环境与依赖
- `ros2 lifecycle`：管理生命周期节点状态

## 6. 坐标变换与运动学

ROS2 常用 `tf2` 处理坐标变换问题：

- 坐标系树（TF tree）
- 坐标变换广播与监听
- 在多传感器、多机器人系统中保持坐标一致性

此外，ROS2 有丰富的运动学和规划工具：

- `Nav2`：移动机器人导航框架
- `MoveIt 2`：机械臂运动规划与控制

## 7. 开源生态与常用框架

ROS2 具有强大的开源社区和生态，部分重要组件包括：

- `Gazebo` / `Ignition Gazebo`：机器人仿真与环境模拟
- `Navigation 2`：移动机器人导航与路径规划
- `MoveIt 2`：机械臂运动规划、碰撞检测和轨迹执行
- `rosbag2`：数据录制与回放
- `launch`：统一的启动系统

## 8. ROS2 的优势

- 支持多语言：C++、Python、Java、Rust 等
- 支持多平台：Linux、Windows、macOS
- 更好的分布式通信与跨网络能力
- 更强的实时性与 QoS 可配置性
- 更灵活的组件化与生命周期管理

## 9. 适合的学习路径

1. 先理解节点、话题、服务、动作的基本概念
2. 从简单发布/订阅入手，写一个最小可运行的节点
3. 了解 ROS2 的命令行工具与 launch 机制
4. 学习 QoS 参数，理解不同场景下的通信策略
5. 进阶学习 `tf2`、`Nav2`、`MoveIt 2`、仿真工具
