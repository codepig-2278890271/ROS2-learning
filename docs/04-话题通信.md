# 话题通信（Topic）

## 什么是话题通信

节点之间怎么互相传递数据？ROS2 最核心的机制就是**话题通信**：

```
节点 A（发布者）                    节点 B（订阅者）
┌─────────────┐                    ┌─────────────┐
│ 摄像头驱动   │  ── /image ──→    │ 图像处理    │
│ Publisher   │     话题           │ Subscriber │
└─────────────┘                    └─────────────┘
```

- 节点 A **发布（publish）** 消息到话题 `/image`
- 节点 B **订阅（subscribe）** 话题 `/image`，自动收到消息
- A 和 B **互相不知道对方的存在**，只和你约好的话题名打交道

这和日常生活中的「订阅公众号」一模一样：作者只负责发文，不关心谁在看；你可以随时关注、随时取关，双方互不干扰。

---

## 核心概念三件套

| 概念 | 是什么 | 类比 |
|------|--------|------|
| **发布者 Publisher** | 生产消息的节点 | 公众号作者，发文到某个专栏 |
| **订阅者 Subscriber** | 消费消息的节点 | 读者，关注某个专栏，推送即收到 |
| **话题 Topic** | 消息的通道，靠**话题名**区分 | 专栏名，作者和读者靠名字对上号 |

关键规则：
- **一个话题可以有多个发布者**（多个作者往同一个专栏投稿）
- **一个话题可以有多个订阅者**（多个读者关注同一个专栏）
- **一个节点可以同时发布和订阅多个话题**（既是作者又是读者）
- 发布者和订阅者**类型必须一致**（Int32 的发 `Int32`，收的也必须用 `Int32`）

## 发布者：最小示例

有求必应，先上代码。`learning_topic/number_publisher.py`：

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import Int32                    # ① 导入消息类型


class NumberPublisher(Node):
    def __init__(self):
        super().__init__('number_publisher')       # ② 节点名
        self.count = 0                             # 计数器，每发一次 +1
        # ③ 创建发布者：话题名 /number，消息类型 Int32，队列深度 10
        self.publisher = self.create_publisher(
            Int32,                                 # 消息类型
            'number',                              # 话题名
            10                                     # QoS 队列深度
        )
        # ④ 定时器：每 2 秒调一次回调，发布一条消息
        self.timer = self.create_timer(2.0, self.timer_callback)

    def timer_callback(self):
        msg = Int32()                              # ⑤ 创建一条 Int32 消息
        msg.data = self.count                      # ⑥ 把计数器的值塞进消息的 data 字段
        self.publisher.publish(msg)                # ⑦ 发布！
        self.get_logger().info(f'发布：{msg.data}')
        self.count += 1


def main():
    rclpy.init()
    node = NumberPublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

### 逐行解释

```python
from std_msgs.msg import Int32
```

`std_msgs` 是 ROS2 自带的标准消息包，里面有一组常用消息类型。`Int32` 就是「一个 32 位整数」——里边只有一个 `data` 字段，往 `data` 里塞你的数字就行。

常用的 `std_msgs` 类型：

| 类型 | 含义 | 字段 |
|------|------|------|
| `Int32` | 32 位整数 | `data` |
| `Int64` | 64 位整数 | `data` |
| `Float32` | 32 位浮点数 | `data` |
| `Float64` | 64 位浮点数 | `data` |
| `String` | 字符串 | `data` |
| `Bool` | 布尔值 | `data` |

> 所有 `std_msgs` 的基础类型都只有一个 `data` 字段，简单直接。

```python
self.publisher = self.create_publisher(Int32, 'number', 10)
```

创建发布者，三个参数：

| 参数 | 值 | 含义 |
|------|-----|------|
| `Int32` | 消息类型 | 这个发布者只发 Int32 格式的消息 |
| `'number'` | 话题名 | `/number`，订阅者通过这个名字找过来 |
| `10` | 队列深度 | 最多暂存 10 条待发送的消息，超过就扔掉最旧的 |

```python
msg = Int32()
msg.data = self.count
self.publisher.publish(msg)
```

三步走：**创建消息 → 赋值 → 发布**。写熟了可以一步：

```python
msg = Int32(data=self.count)   # 直接在构造里赋值，和上面三行等价
self.publisher.publish(msg)
```

## 订阅者：最小示例

`learning_topic/number_subscriber.py`：

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import Int32


class NumberSubscriber(Node):
    def __init__(self):
        super().__init__('number_subscriber')         # ① 节点名
        # ② 创建订阅者：话题名 /number，消息类型 Int32，收到消息调 listener_callback
        self.subscription = self.create_subscription(
            Int32,                                    # 消息类型，必须和发布者一致
            'number',                                 # 话题名，必须和发布者一致
            self.listener_callback,                   # ③ 回调函数
            10                                        # QoS 队列深度
        )

    def listener_callback(self, msg):                 # ④ 收到消息时自动调用
        squared = msg.data ** 2                       # 收到数字后求平方
        self.get_logger().info(
            f'收到：{msg.data} → 平方后：{squared}'
        )


def main():
    rclpy.init()
    node = NumberSubscriber()
    rclpy.spin(node)                                  # ⑤ spin 循环里等待消息
    node.destroy_node()
    rclpy.shutdown()
```

### 和发布者的关键区别

```
发布者：主动发                  订阅者：被动收
create_publisher()             create_subscription()
publish(msg)                   回调函数自动被调用
定时器触发                      spin() 循环里等待
```

订阅者**不需要定时器**。`rclpy.spin(node)` 会一直跑，每当话题上有新消息到达，ROS2 就自动调用你的 `listener_callback`，把消息对象作为参数 `msg` 传进去。你只负责写「收到消息后干什么」，不用管「什么时候收」。

### 回调函数的参数

```python
def listener_callback(self, msg):
```

- `msg` 就是发布者 `publish()` 出去的那个消息对象
- `msg.data` 就是你发的那个 `Int32` 的值
- 消息类型不同，可用的字段也不同（Int32 只有 `data`，后面会碰到有 `x, y, z` 的 Point）

## 运行演示

先确保编译并 source：

```bash
cd ~/ros2_ws
colcon build --packages-select learning_topic
source install/setup.bash
```

打开**三个终端**：

```
终端1（发布者）：         终端2（订阅者）：             终端3（查看话题）：
$ ros2 run                $ ros2 run                    $ ros2 topic list
  learning_topic            learning_topic              /number
  number_publisher          number_subscriber            /parameter_events
                                                          /rosout
发布：0                   收到：0 → 平方后：0
发布：1                   收到：1 → 平方后：1           $ ros2 topic echo /number
发布：2                   收到：2 → 平方后：4           data: 3
发布：3                   收到：3 → 平方后：9           ---
...                       ...                           data: 4

```

- 终端 1 和 2 通过 `/number` 话题联通，互不感知
- 终端 3 用 `ros2 topic echo` 旁观——它本身也是一个隐式的订阅者，帮你 debug 用
- 终端 2 求平方是在**本地**算的，发布者并不知道订阅者做了什么处理

## 消息类型不止 Int32

`std_msgs` 只能传单个数据，实际项目中消息通常是**复合类型**，比如 `geometry_msgs/Point`：

```python
from geometry_msgs.msg import Point

pose = Point()
pose.x = 1.5
pose.y = 2.0
pose.z = 0.0
```

一个 `Point` 消息同时带 x、y、z 三个坐标。ROS2 自带了很多复合消息类型，常用的：

| 类型 | 来自 | 字段 |
|------|------|------|
| `Point` | `geometry_msgs.msg` | `x, y, z`（float64） |
| `Twist` | `geometry_msgs.msg` | `linear.x/y/z` + `angular.x/y/z`（速度+角速度） |
| `Pose` | `geometry_msgs.msg` | `position`（Point）+ `orientation`（Quaternion） |
| `Image` | `sensor_msgs.msg` | `width, height, data` 等（图片） |
| `LaserScan` | `sensor_msgs.msg` | `ranges, angle_min, angle_max` 等（激光雷达扫描） |

## 常用命令

```bash
# 话题列表与信息
ros2 topic list                    # 列出所有话题
ros2 topic list -t                 # 列出话题 + 消息类型
ros2 topic info /number            # 查看话题详情（类型、发布/订阅者数量）

# 查看话题内容
ros2 topic echo /number            # 实时打印话题上流通的消息（debug 神器）
ros2 topic echo /robot_pose        # 看巡逻机器人的实时位置
ros2 topic echo --once /number     # 只看一条就退出
ros2 topic echo --csv /number      # 输出 CSV 格式，方便导给 Python/Excel

# 手动发布消息（测试用）
ros2 topic pub /number std_msgs/msg/Int32 "data: 42"         # 手动发一条
ros2 topic pub -r 2 /number std_msgs/msg/Int32 "data: 42"    # 以 2Hz 频率持续发
ros2 topic pub --once /emergency_stop std_msgs/msg/Int32 "data: 1"  # 发一条急停

# 查看频率
ros2 topic hz /number              # 查看话题的发布频率（Hz）
```

> `ros2 topic pub` 是测试利器。拿它手动发消息，不用写发布者也能调订阅者。

## 关键设计决策：为什么话题名要在代码里写死

你可能注意到了：`create_publisher(Int32, 'number', 10)` 里话题名 `'number'` 是写死在代码里的。那想换话题名怎么办？

这是有意为之：**发布者和订阅者通过约定好的话题名来匹配，这是一种「靠名字解耦」的设计。** 实际项目中可以加参数化：

```python
self.declare_parameter('topic_name', 'number')
topic = self.get_parameter('topic_name').value
self.pub = self.create_publisher(Int32, topic, 10)
```

然后运行时重映射：

```bash
ros2 run learning_topic number_publisher --ros-args -r number:=my_custom_number
```

> 初学阶段写死就好，省心。后面做复用时再考虑参数化。

## 总结

```
话题通信 = 发布者 + 订阅者 + 话题名

发布者三要素：
  ① create_publisher(消息类型, 话题名, 队列深度)
  ② 创建消息对象 → 给字段赋值
  ③ publish(msg) 发布

订阅者三要素：
  ① create_subscription(消息类型, 话题名, 回调函数, 队列深度)
  ② 回调函数 listener_callback(self, msg)
  ③ rclpy.spin(node) 循环等待

核心规则：
  - 消息类型必须一致（Int32 ↔ Int32）
  - 话题名必须一致（'number' ↔ 'number'）
  - 发布者和订阅者互不感知，靠话题名连接
  - 一个节点可以同时发布和订阅多个话题
  - ros2 topic echo /话题名 是最常用的 debug 手段
```

| 对比 | 发布者 Publisher | 订阅者 Subscriber |
|------|-----------------|-------------------|
| 角色 | 生产消息 | 消费消息 |
| 创建方式 | `create_publisher()` | `create_subscription()` |
| 触发方式 | 主动调用 `publish()` | 被动回调 `listener_callback()` |
| 需要定时器？ | 通常需要（定时发） | 不需要（等消息来） |

下一步：话题是最基础的通信方式，但有个限制——它是**单向的**。你想问相机节点「你分辨率多少？」，光靠话题不行（你得先知道它发了多少分辨率的消息，再订阅它）。这时候需要**服务（Service）**，一问一答，下一章继续。
