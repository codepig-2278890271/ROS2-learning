# 参数（Parameter）

## 为什么需要参数

学完了三大通信模型——话题、服务、动作——你已经能让节点之间互相通信了。但现在有个问题：

**你的节点全是「写死」的。**发布频率 1Hz、相机分辨率 640×480、速度上限 0.5m/s……全是硬编码在代码里的常量。想改？改代码重编译。

参数就是用来解决这个的：**节点的运行时配置，改完立刻生效，不用重新编译。**

```
┌─────────────────────────────────┐
│  节点                            │
│                                 │
│  camera_width = 640     ← 参数   │
│  camera_height = 480    ← 参数   │
│  frame_rate = 30        ← 参数   │
│  exposure = 0.5         ← 参数   │
│                                 │
│  「这几个值用户随时可以改」        │
└─────────────────────────────────┘
```

参数是节点的**配置层**——和通信是两码事。通信是节点间的对话，参数是节点自己的设置。

---

## 参数 vs 话题/服务/动作

| | 话题/服务/动作 | 参数 |
|------|--------------|------|
| 作用 | **节点间通信** | **节点配置** |
| 方向 | 节点 → 节点 | 外部 → 节点 |
| 持久化 | 不保存 | 可以存到文件 |
| 改完要不要重编译 | 不需要 | 不需要 |
| 类比 | 打电话、发信息 | 设置菜单里的选项 |

---

## 参数的本质：键值对

ROS2 参数就是 `key = value`，支持这些类型：

| 类型 | 示例 | 说明 |
|------|------|------|
| `bool` | `True`, `False` | 开关 |
| `int` | `30`, `640` | 整数 |
| `double` | `0.5`, `3.14` | 浮点数 |
| `string` | `"hello"` | 字符串 |
| `int[]` | `[1, 2, 3]` | 整数数组 |
| `double[]` | `[0.1, 0.2]` | 浮点数组 |
| `string[]` | `["a", "b"]` | 字符串数组 |

> 不支持自定义 msg 类型——参数是简单数据，不是消息。

## 声明参数并设置默认值

```python
import rclpy
from rclpy.node import Node


class MyNode(Node):

    def __init__(self):
        super().__init__('my_node')

        # ① 声明参数（键 = 默认值）
        self.declare_parameter('camera_width', 640)
        self.declare_parameter('camera_height', 480)
        self.declare_parameter('frame_rate', 30)
        self.declare_parameter('topic_name', '/image_raw')

        self.get_logger().info('参数已声明')
```

`declare_parameter('键', 默认值)` 做了两件事：
- 如果用户没设置这个参数 → 用默认值
- 如果用户设置了 → 用设置的值覆盖

---

## 读取参数

```python
# 方式一：get_parameter() —— 返回 Parameter 对象
param = self.get_parameter('camera_width')
width = param.value                         # 640

# 方式二：get_parameter('键').get_parameter_value() —— 链式调用
height = self.get_parameter('camera_height').get_parameter_value().integer_value

# 方式三（推荐）：用 descriptor 一次性取出值
width = self.get_parameter('camera_width').value  # 最简洁
```

### 针对不同类型的取值方法

| get_parameter_value() 的方法 | 对应类型 |
|------------------------------|---------|
| `.bool_value` | bool |
| `.integer_value` | int |
| `.double_value` | double |
| `.string_value` | string |
| `.bool_array_value` | bool[] |
| `.integer_array_value` | int[] |
| `.double_array_value` | double[] |
| `.string_array_value` | string[] |

> 日常开发用 `.value` 就够了，自动推断类型。

## 典型用法：让参数控制节点行为

比如一个定时发布器，发布频率由参数决定：

```python
class PeriodicPublisher(Node):

    def __init__(self):
        super().__init__('periodic_publisher')

        # 声明参数
        self.declare_parameter('publish_rate', 1.0)   # Hz，默认每秒 1 次
        self.declare_parameter('message_text', 'hello')

        # 读取参数
        rate = self.get_parameter('publish_rate').value
        text = self.get_parameter('message_text').value

        # 用参数创建发布器
        self._publisher = self.create_publisher(String, 'chatter', 10)
        self._timer = self.create_timer(1.0 / rate, self.on_timer)
        self._text = text

        self.get_logger().info(f'启动: rate={rate}Hz, text="{text}"')
```

现在想改发布频率？不用改代码，一行命令搞定。

## 命令行调试：ros2 param

```bash
# 启动节点
ros2 run learning_parameter my_node

# 查看节点有哪些参数
ros2 param list
# 输出: /my_node:
#         camera_height
#         camera_width
#         frame_rate
#         topic_name

# 看某个参数的值
ros2 param get /my_node camera_width
# 输出: Integer value is: 640

# 改参数（运行时修改！）
ros2 param set /my_node camera_width 1280
# 输出: Set parameter successful

# 确认改成功
ros2 param get /my_node camera_width
# 输出: Integer value is: 1280
```

**这就是参数的魅力**：节点在跑，参数说改就改，不用停节点、不用重编译。

### ros2 param set 的类型推断

```bash
ros2 param set /my_node enable_debug true      # → bool
ros2 param set /my_node max_retry 3            # → int
ros2 param set /my_node speed 0.75             # → double
ros2 param set /my_node name "robot_1"          # → string
ros2 param set /my_node ports [1, 2, 3]        # → int[]
```

## 监听参数变化：add_on_set_parameters_callback

改参数是一回事，**感知**参数被改了是另一回事。订阅参数变化：

```python
from rclpy.parameter import Parameter
from rcl_interfaces.msg import SetParametersResult


class ReactiveNode(Node):

    def __init__(self):
        super().__init__('reactive_node')

        self.declare_parameter('brightness', 0.5)
        self.declare_parameter('enable_auto_exposure', True)

        # 注册回调：参数被 set 时触发
        self.add_on_set_parameters_callback(self.on_param_changed)

        self.get_logger().info(f'初始亮度: {self.get_parameter("brightness").value}')

    def on_param_changed(self, params: list[Parameter]):
        """参数被修改时自动调用。params 是被改的参数列表。"""
        result = SetParametersResult(successful=True)

        for param in params:
            if param.name == 'brightness':
                # 可以在这里做校验
                if not 0.0 <= param.value <= 1.0:
                    result.successful = False
                    result.reason = '亮度必须在 0.0 ~ 1.0 之间'
                    return result
                self.get_logger().info(f'亮度调到: {param.value}')

            elif param.name == 'enable_auto_exposure':
                self.get_logger().info(f'自动曝光: {"开" if param.value else "关"}')

        return result  # 必须返回 SetParametersResult
```

### 回调关键点

```
外部 set 参数
    │
    ▼
add_on_set_parameters_callback 回调
    │
    ├─ 校验参数值
    │     ├─ 合法 → 更新内部状态，返回 (successful=True)
    │     └─ 不合法 → 拒绝修改，返回 (successful=False, reason="…")
    │
    └─ 返回 SetParametersResult  —— 告诉框架「改好了」或「改不了」
```

- 返回值 `successful=False` 时，参数**不会被修改**，保持原值
- `reason` 字符串会显示给调用者（比如 `ros2 param set` 会打印原因）

## 从 YAML 文件加载参数

一个个 `ros2 param set` 太累，参数多了应该用文件：

```yaml
# config/camera.yaml
/my_camera_node:
  ros__parameters:
    width: 1280
    height: 720
    frame_rate: 30
    auto_exposure: true
    exposure_time: 0.05
```

启动时指定文件：

```bash
ros2 run learning_parameter camera_node --ros-args --params-file config/camera.yaml
```

要对多个节点配置参数：

```yaml
# config/app.yaml
/my_camera_node:
  ros__parameters:
    width: 1280
    height: 720

/my_detector_node:
  ros__parameters:
    model_path: "/models/yolo.pt"
    confidence_threshold: 0.6
```

## declare_parameter 的完整签名

```python
self.declare_parameter(
    'param_name',        # 参数名
    default_value,       # 默认值
    descriptor=ParameterDescriptor(
        description='摄像头的宽度（像素）',      # 描述（给人类看的）
        type=ParameterType.PARAMETER_INTEGER,    # 约束类型
        integer_range=[IntegerRange(from_value=1, to_value=4096, step=1)],  # 约束范围
    )
)
```

`ParameterDescriptor` 是可选的，但它让你给参数加**元数据**——其他工具（比如 GUI 面板）可以读取描述信息，给用户更好的体验。

## 总结

```
参数 = 节点的运行时配置

三个核心操作：
  ① declare_parameter('键', 默认值)        声明参数
  ② get_parameter('键').value              读取参数
  ③ set_parameter / add_on_set_parameters_callback   修改 + 监听

CLI 三件套：
  ros2 param list    —— 看有哪些参数
  ros2 param get     —— 看参数值
  ros2 param set     —— 改参数（运行时生效）

参数 vs 通信：
  - 通信 = 节点间的对话（话题/服务/动作）
  - 参数 = 节点自己的设置（键值对，可运行时改）
```

| | 话题 Topic | 服务 Service | 动作 Action | 参数 Parameter |
|------|-----------|-------------|-------------|---------------|
| 用途 | 单向数据流 | 请求-响应 | 长时间任务 | **节点配置** |
| 方向 | pub → sub | cli ⇄ srv | cli ⇄ srv | **外部 → 节点** |
| 持久化 | 无 | 无 | 无 | **可 YAML 文件** |
| CLI | `ros2 topic` | `ros2 service` | `ros2 action` | `ros2 param` |
| 阻塞 | 不阻塞 | 可阻塞 | 异步 | 即时 |

学了参数，你的节点终于不是「写死」的了。配合 YAML 文件，一套代码配多套配置——跑仿真一套、跑真机另一套，互相不打架。
