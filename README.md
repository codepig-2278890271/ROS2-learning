## 关键理解

- **话题名必须一致**：发布者的 `create_publisher(..., '话题名', ...)` 和订阅者的 `create_subscription(..., '话题名', ...)` 使用**同一个话题名**才能通信。这就好比对讲机——双方必须在同一个频道上才能听到对方。话题名是发布者和订阅者之间唯一的约定，它们互相不知道对方存在，全靠名字对号入座。
- **话题是广播，服务是电话**：话题通信是单向的，发布者只管发，不知道谁在收；服务通信是双向的，客户端发请求、服务端返回响应，一问一答。传感器数据流用话题，查询/指令用服务。
